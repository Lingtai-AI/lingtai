README first-time install troubleshooting

结论：本次改动把「首次安装排障」内容带回 README.md：在 Install in detail 的 Homebrew 安装/升级命令之后、Architecture 之前，新增 ### First-time install troubleshooting 小节，覆盖三类新用户卡点 —— 没有 brew、 WSL/Ubuntu 缺 build-essential 等先决依赖、以及完全没有 Homebrew 时的源码构建路径。 不改变发布语义：Homebrew 仍是推荐安装方式，pip install lingtai 仍然只是 kernel 开发/诊断用途。

为什么要做

早期 README 的 Quick start 里曾有过这块内容（见 git 历史 af2ba57 "docs: add Linux/WSL install steps to all READMEs" 与 c58e09e "clean quickstart — Homebrew install collapsible"），后来在 72a5c71 "rewrite README product narrative" 的重写中被移除。结果是：Linux/WSL 新用户跑 brew install lingtai-ai/lingtai/lingtai-tui 时，要么 brew: command not found， 要么 Homebrew on Linux 因缺 gcc 而编译失败，README 里没有任何指引。Jason 要求把这块内容恢复， 放在安装详情之后、保持产品 README 的简洁体例。

新小节内容（README.md, Install in detail 内）

卡点给出的命令 / 指引
没有 brew（macOS 或 Linux） 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
并提示安装结束后按 installer 输出的 eval 行把 brew 加进 PATH（或开新终端）。
WSL / Ubuntu / Debian 
sudo apt update
sudo apt install build-essential curl git ca-certificates
先装先决依赖，再装 Homebrew，再 brew install。相比历史版本（只有 build-essential）， 按任务要求补上了 curl git ca-certificates —— 这是 Homebrew installer 自身的依赖。
完全没有 Homebrew 指向同一节内已有的 From source 小节 （anchor 链接 #from-source），并点名先决依赖：Go、make、portal 需要 Node.js/npm。

其他改动

  • Quick start（"Start in three commands"）下方的指路句同步更新，加上 "first-time install troubleshooting (missing brew, WSL prerequisites)"， 让新用户从页首就能跳到这一节。
  • 页尾已有的 ## Troubleshooting（运行期排障）保持不动 —— 新小节只管「装不上」， 老节继续管「装上了但不工作」。两者不重叠。

位置选择

放在 ### Homebrew (recommended) 与 ### From source 之间： 紧跟 brew install/upgrade 命令（用户失败的第一现场），又在 ## Architecture 之前， 符合任务给出的位置约束。作为 ### 子节挂在 Install in detail 下，不打断顶层叙事节奏。
