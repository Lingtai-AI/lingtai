install.sh no-login source download repair

Follow-up after PR #305: make the source-build helper avoid GitHub account prompts.

Conclusion

install.sh should be non-interactive. Even though LingTai is public, using git clone https://github.com/... inside an installer can trigger local git credential helpers or network/proxy auth prompts on some machines. The fix is to download GitHub's public source archive via codeload.github.com instead of cloning.

What changed

  • Replaced git clone with curl -fL https://codeload.github.com/Lingtai-AI/lingtai/tar.gz/<ref>.
  • Removed the hard dependency on git for normal installer operation.
  • Added explicit curl and tar dependency checks.
  • Kept Homebrew as the primary distribution path; this remains a source-build helper.
  • Kept the writable install-dir fallback from #239/#240/#305.
