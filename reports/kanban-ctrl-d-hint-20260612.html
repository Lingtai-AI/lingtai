/kanban 顶部加入 Ctrl+D 详情提示

本小改采纳 ktwu01 #289 背后的可发现性问题，但不新增顶层 /context 命令：既有入口仍为 /kanban，详情页仍由 Ctrl+D 打开。

结论

可合并。 改动范围小，未改变 slash-command 路由、未新增命令，只把 Ctrl+D 提示从右下角弱提示提升为 /kanban 标题分隔线下方的醒目提示。

TUI-onlyi18n en/zh/wen测试覆盖无外部副作用

为什么不收 /context

  • /context 只是 /kanban → Ctrl+D 的别名，不提供新能力。
  • 新增顶层 slash command 会扩大命令面，降低命令列表清晰度。
  • 真正问题是用户不容易发现 Ctrl+D，因此应修提示，而非加别名。

改动内容

  • tui/internal/tui/props.go：在 /kanban 非详情模式标题下方渲染醒目的 ctrl+d open context detail breakdown callout；详情模式中隐藏该 callout。
  • tui/i18n/{en,zh,wen}.json：加入 props.ctrl_d_hint，并将原 footer 中 detail 文案扩成“context detail / 上下文详情 / 语境详”。
  • tui/internal/tui/props_test.go：新增测试，确认普通 /kanban 视图显示 callout，详情视图不重复显示，同时 footer 仍保留 ctrl+d 提示。

验证

cd tui && go test ./internal/tui ./i18n -count=1
cd tui && go test ./... -count=1
git diff --check

三项均已通过。全量 go test ./... 亦通过；测试生成的 tui/internal/tui/logs/ 已清除。

风险与取舍

  • 顶部多占 1 行视口高度，以换取可发现性；这是 Jason 明确要求的“上面显眼”提示。
  • 不触碰命令 palette，不影响用户已知的 /kanban 与 Ctrl+D 行为。
  • 不解决 #237/#299/#302 的较大功能诉求；这些已另行 triage。
