Preset Editor API Mode Controls

Local TUI PR prep · branch fix/preset-editor-wire-api-service-tier · commit 757c9e54 · base origin/main@f976aecb

TL;DR

Conclusion: this local commit exposes the backend OpenAI-compatible controls in the TUI preset editor: API mode for wire_api, and Fast mode for service_tier="fast". It also preserves provider-specific non-empty service_tier values when users save unrelated preset edits.

Baseline

  • Before: users could not select OpenAI-compatible responses vs chat_completions from the preset editor.
  • Before: service tier was Codex-shaped in the editor and not presented as a general OpenAI-compatible fast/default control.
  • Repo state used for prep: clean worktree from origin/main@f976aecb; original v0.10.5 dirty patch was preserved as <agent-artifacts>/lingtai-custom-api-pr-prep/tui-custom-api-v0.10.5-20260709.patch.

What Was Done

API mode

Preset editor now renders auto, chat_completions, and responses. Explicit modes persist manifest.llm.wire_api; auto removes it.

Fast mode

Preset editor now renders default and fast for all providers. Selecting fast writes service_tier="fast"; selecting default removes the field.

Compatibility

Legacy response flags display as responses and are cleared when the user chooses an explicit API mode.

Preservation

Existing non-empty provider-specific service tiers are preserved on unrelated commits, even if the simplified radio strip displays default.

Risks / Decisions

  • Hidden provider-specific tiers: the radio strip only shows default/fast, so an existing non-standard tier is preserved but visually appears as default until the user cycles it.
  • Explicit auto is omitted: selecting API mode auto deletes wire_api rather than persisting wire_api="auto". This keeps presets minimal but means explicit auto does not round-trip textually.
  • Full test suite: OAuth loopback timing failure remains a residual risk outside the changed preset editor surface.
  • Decision: expose only the common fast/default UI, while preserving arbitrary backend-supported tier strings unless the user explicitly changes that row.
