Issue #178 — Scientific Cultivation (科学修仙) guardrail skill

Conclusion: this docs-only change adds a single self-contained preset utility skill — tui/internal/preset/skills/scientific-cultivation/SKILL.md — that tells an agent how to gently protect the human behind a long LingTai session: notice fatigue, seal the worksite, keep safe background jobs running, and offer one rest without shame or nagging. No runtime, config, or release changes.

护元而不误事，守炉而不添火 — protect the human's vital essence, do not delay the work; tend the furnace, do not add fire.

Why this skill exists

LingTai runs long, multi-agent, high-output workflows. The agents do not tire; the human does. When the human gets pulled into a long, excited loop, LingTai should help them rest without stopping the agent network. The covenant already frames the human's work as cultivation and reminds that "cultivation is not the purpose — solving the struggles of others is." A depleted human aids no one, so rest is reframed here as scientific cultivation rather than a lapse in discipline.

The cultivation loop

1. Notice    — a soft fatigue / late-night / "one more" signal appears.
2. Seal      — capture the current worksite: done / pending / next entry point.
3. Keep      — confirm safe background jobs continue; nothing is lost.
4. Offer     — suggest ONE gentle restorative (water / eyes / movement / sleep).
5. Defer     — accept the human's choice without argument or repetition.

The three commitments (in priority order)

CommitmentMeaning
护元 — protect vital essenceNotice fatigue, offer rest. Wins when commitments conflict.
不误事 — do not delay the workSeal the worksite and keep safe background jobs running so nothing is lost when the human steps away.
不添火 — do not add fireNever inject urgency, new scope, shame, or stimulation to keep a tired human at the keyboard.

Design choices worth noting

  • Tone over mechanism. The skill changes how an agent talks to the human, nothing else. Most of the file is anti-nagging guidance, because tone is where this kind of guardrail fails.
  • Offer once, then defer. A "no" is respected immediately; the skill re-offers only on a genuinely new and stronger signal, never on a timer.
  • Never stop the network to force a break. Sealing the worksite and keeping safe jobs running is what makes rest cheap rather than a sacrifice — 不误事 in practice.
  • Soft signals, not surveillance. The agent rarely has the clock or biometrics; fatigue cues are treated as soft and private, never broadcast to other agents.

Integration — why no Go change is needed

Preset skills are embedded via //go:embed skills in tui/internal/preset/preset.go and extracted on startup by a fs.WalkDir over the whole skills/ tree (PopulateBundledLibrary). A new skill directory is discovered automatically; there is no explicit skill registry or count to update. The embed comment warns only against dot-prefixed files — this skill ships none.

Risk assessment

Low risk. Documentation/skill text only. No runtime code, build scripts, dependency manifests, configuration, secrets, or release tooling are touched. The skill explicitly disclaims side effects: it never stops or reconfigures the agent network on its own, and pausing a job remains a human-initiated action.

Validation

  • git diff --check — no whitespace errors.
  • YAML frontmatter present with name / description / version / tags, matching the convention used by existing single-file skills (minimax-cli, lingtai-tui-anatomy).
  • Skill directory scientific-cultivation/ sits alongside peers under tui/internal/preset/skills/ and is covered by the existing //go:embed skills + fs.WalkDir extraction path.
  • No existing preset test asserts an exact skill count, so the new directory does not break swiss_knife_populate_test.go or its siblings (they verify named skills survive, not a total).

Caveats

  • Behavioral, not enforced. This is prompt guidance an agent reads — it nudges behavior, it does not technically prevent late-night work.
  • The reference implementation (9s5bz2jvd2-lang/scientific-cultivation-skill) informed the concept; this version is rewritten LingTai-native, leaning on the covenant's cultivation/furnace metaphor.
